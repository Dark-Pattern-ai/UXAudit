# 다크패턴 탐지 데이터셋 라벨링 가이드 v1.1

## 1. 기본 규칙

**적용 범위:** 금융 앱/웹뿐 아니라 쇼핑몰, 구독 서비스, 배달 앱, OTT, 통신사 앱 등 **온라인에서 상품·서비스를 판매하는 모든 UI**를 대상으로 합니다.

**라벨링 단위:** 스크린샷 1장 = 라벨 1개 (가장 눈에 띄는 다크패턴 1개를 primary로 선택)

**복수 패턴이 보이는 경우:** `secondary_label`에 두 번째 유형 id를 기입 (없으면 null)

**판단 순서:** 스크린샷을 봤을 때 "일반 사용자가 가장 먼저 속거나 불편을 느낄 요소"가 primary_label이다.

## 2. 라벨 체계

금융위 가이드라인의 15개 유형을 기반으로 하되, 온라인 커머스 전반에 적용할 수 있도록 판단 기준을 확장합니다.

| label id | category | 유형 이름 | 한 줄 판단 기준 |
| --- | --- | --- | --- |
| 0 | NORMAL | 정상 | 아래 1~15 어디에도 해당 안 됨 |
| 1 | MISLEADING | 설명절차의 과도한 축약 | 가입/구매/계약 과정에서 설명 단계가 빠져 있어서 사용자가 뭘 동의하는지 모르고 넘어가게 됨 |
| 2 | MISLEADING | 속임수 질문 | 질문 문장이 이중부정이거나 헷갈려서 의도와 반대로 답하기 쉬움 |
| 3 | MISLEADING | 잘못된 계층구조 | 사업자에게 유리한 버튼이 크거나 진하고, 사용자에게 유리한 버튼이 작거나 흐림 |
| 4 | MISLEADING | 특정 옵션 사전선택 | 체크박스나 토글이 이미 켜져 있어서 사용자가 모르고 넘어가면 사업자에게 유리한 옵션이 적용됨 (예: 부가 서비스 자동 추가, 마케팅 수신 동의, 정기구독 자동 선택 등) |
| 5 | MISLEADING | 허위광고 및 기만적 유인행위 | 근거 없는 수치, 과장된 할인율/수익률, "최저가 보장" 같은 오해 유발 표현이 보임 |
| 6 | OBSTRUCTING | 취소·해지·탈퇴 방해 | 가입/구매는 간단한데 해지/환불/탈퇴 경로가 없거나 전화 연결을 요구하는 등 명백히 더 복잡함 |
| 7 | OBSTRUCTING | 숨겨진 정보 | 위약금, 자동갱신 조건, 배송비, 수수료 등 중요 정보가 작은 글씨·흐린 색상·스크롤 아래 등에 숨겨져 있음 |
| 8 | OBSTRUCTING | 가격비교 방해 | 가격·이율·수수료 등을 다른 상품과 비교하기 어렵게 표시 (단위 다름, 조건 다름, 나열 방식 다름) |
| 9 | OBSTRUCTING | 클릭 피로감 유발 | 사용자에게 유리한 선택을 하려면 해제/클릭을 비정상적으로 많이 해야 함 |
| 10 | PRESSURING | 계약과정 중 기습적 광고 | 가입/구매/결제 절차 도중 갑자기 관련 없는 다른 상품/서비스 광고 팝업이 뜸 |
| 11 | PRESSURING | 반복 간섭 | 사용자가 "안 함"을 선택했는데 같은 요청을 다시 팝업으로 띄움 |
| 12 | PRESSURING | 감정적 언어 사용 | "놓치면 후회", "지금 안 사면 품절", "부담스러우세요?" 등 감정을 자극하는 문구로 행동을 유도 |
| 13 | PRESSURING | 감각 조작 | 특정 요소가 깜빡이거나 색상 변화·애니메이션으로 시선을 강제 유도 |
| 14 | PRESSURING | 다른 소비자의 활동 알림 | "최근 N명이 구매", "지금 N명이 보는 중", "장바구니에 N개 담김" 등 타인 행동을 보여주며 압박 |
| 15 | EXPLOITING | 순차공개 가격책정 | 첫 화면에서 최저가/최고 이율만 보여주고, 진행할수록 배송비·수수료·조건 미충족 등 실제 비용이 드러남 |

## 3. 헷갈리는 경계 사례 판단표

| 상황 | 이거 아닌가? | → 이걸로 판정 | 이유 |
| --- | --- | --- | --- |
| 동의/구매 버튼만 크고 화려, 거절/취소 버튼은 흐림 | ⑥ 취소방해? | → **③ 잘못된 계층구조** | 버튼이 화면에 둘 다 존재하긴 함. 시각적 차이로 유도하는 것이므로 오도형 |
| 거절/해지/환불 버튼이 화면에 아예 없음 | ③ 잘못된 계층구조? | → **⑥ 취소·해지 방해** | 선택지 자체가 제거된 것이므로 방해형 |
| 체크박스가 미리 켜져 있고 + 감정적 문구 동반 | ⑫ 감정적 언어? | → **④ 특정 옵션 사전선택** (primary), ⑫ (secondary) | 사전선택이 직접적 행동 왜곡, 감정 문구는 보조적 |
| "지금 N명 구매" + 최저가만 표시 | ⑮ 순차공개? | → **⑭ 다른 소비자 활동 알림** (primary), ⑮ (secondary) | N명 표시가 화면에서 더 눈에 띄면 ⑭, 가격 왜곡이 핵심이면 ⑮ |
| 신청 안 함 눌렀는데 팝업 다시 뜸 | ⑩ 기습적 광고? | → **⑪ 반복 간섭** | 같은 요청을 반복하는 것이 핵심이므로 반복 간섭 |
| 가입/결제 중 갑자기 전혀 다른 상품 팝업 | ⑪ 반복 간섭? | → **⑩ 기습적 광고** | 이전에 거절한 적 없는 새 상품 광고가 뜬 것이므로 기습적 광고 |
| 중요 정보가 작은 글씨 + 비교도 어려움 | ⑧ 가격비교 방해? | → **⑦ 숨겨진 정보** (primary), ⑧ (secondary) | 정보 자체가 은폐된 게 더 근본적 문제 |
| 쇼핑몰에서 "오늘만 이 가격" + 카운트다운 타이머 | ⑬ 감각 조작? | → **⑫ 감정적 언어 사용** (primary), ⑬ (secondary) | 문구가 핵심 압박 수단이면 ⑫, 타이머 애니메이션이 핵심이면 ⑬ |
| 구독 해지 과정에서 "해지하면 혜택을 모두 잃습니다" 팝업 | ⑥ 취소방해? ⑫ 감정적 언어? | → **⑥ 취소·해지 방해** (primary), ⑫ (secondary) | 해지 흐름을 끊는 것이 핵심 행위, 감정적 문구는 보조 수단 |

## 4. NORMAL 판정 기준

아래에 해당하면 다크패턴처럼 보여도 **0 (NORMAL)** 로 처리합니다.

- 법적으로 필수인 약관 동의 체크박스가 미체크 상태로 제공되고, 사용자가 직접 체크해야 진행되는 경우 → 정상
- 버튼 크기나 색상에 차이가 있지만 "확인 / 취소"처럼 일반적인 UI 관행 수준인 경우 → 정상
- 상품 정보 페이지에서 조건별 가격 범위(최저~최고)를 함께 보여주는 경우 → 정상 (가격비교 방해 아님)
- 법령에 따른 위험 고지·환불 규정이 별도 영역에 명확히 구분되어 있는 경우 → 정상
- 쇼핑몰의 일반적인 "추천 상품" 영역이 상품 상세 하단에 있는 경우 → 정상 (기습적 광고 아님, 결제 흐름 중간이 아님)
- "N명이 이 상품을 봤습니다"가 단순 통계 정보로 제공되고, 구매 압박 맥락이 아닌 경우 → 정상

## 5. GPT-4o Vision 라벨링 프롬프트

200장을 GPT-4o에 넣을 때 아래 프롬프트를 그대로 복붙해서 씁니다.

```
너는 온라인 서비스 UI 다크패턴 라벨링 전문가야.
첨부된 스크린샷은 금융 앱, 쇼핑몰, 구독 서비스, 배달 앱 등
온라인에서 상품·서비스를 판매하는 화면이야.
아래 16개 라벨 중 해당하는 것을 골라줘.

[라벨 목록]
0  NORMAL         정상 (다크패턴 없음)
1  MISLEADING     설명절차의 과도한 축약
2  MISLEADING     속임수 질문
3  MISLEADING     잘못된 계층구조
4  MISLEADING     특정 옵션 사전선택
5  MISLEADING     허위광고 및 기만적 유인행위
6  OBSTRUCTING    취소·해지·탈퇴 방해
7  OBSTRUCTING    숨겨진 정보
8  OBSTRUCTING    가격비교 방해
9  OBSTRUCTING    클릭 피로감 유발
10 PRESSURING     계약과정 중 기습적 광고
11 PRESSURING     반복 간섭
12 PRESSURING     감정적 언어 사용
13 PRESSURING     감각 조작
14 PRESSURING     다른 소비자의 활동 알림
15 EXPLOITING     순차공개 가격책정

[판단 규칙]
- 다크패턴이 없으면 반드시 0을 선택해.
- 다크패턴이 여러 개 보이면, 사용자가 가장 먼저 속거나 불편을 느낄 요소를 primary로 골라.
- 두 번째로 눈에 띄는 패턴이 있으면 secondary에 넣어. 없으면 null.
- 감각 조작(13)은 정적 스크린샷으로는 판단이 어려우니, 깜빡임/애니메이션 흔적이 보이지 않으면 선택하지 마.
- 금융 서비스뿐 아니라 쇼핑몰, 구독, 배달 등 모든 온라인 커머스에 동일한 기준을 적용해.
- 다크패턴인지 아닌지 판단이 애매하면, 가장 가까운 라벨을 붙이되 confidence를 0.5 이하로 낮게 줘. reason에 "판단 보류 - 애매함"이라고 명시해.

[NORMAL로 판정하는 경우]
- 법적 필수 약관 동의 체크박스가 미체크 상태로 제공되는 경우
- 확인/취소 버튼의 크기·색상 차이가 일반적인 UI 관행 수준인 경우
- 상품 상세 하단의 일반적인 추천 상품 영역 (결제 흐름 중간이 아닌 경우)
- 단순 통계성 정보 ("N명이 봤습니다")가 구매 압박 맥락 없이 제공되는 경우

[경계 사례 규칙]
- 버튼이 둘 다 있는데 시각 차이로 유도 → ③ 잘못된 계층구조
- 해지/거절 버튼이 화면에 아예 없음 → ⑥ 취소·해지 방해
- "안 함" 선택 후 같은 요청 재팝업 → ⑪ 반복 간섭
- 결제 중 관련 없는 새 상품 팝업 → ⑩ 기습적 광고

[출력 형식 - 반드시 JSON 한 줄로만 답해]
{"file": "파일명", "primary_label": 숫자, "primary_category": "카테고리", "secondary_label": 숫자 또는 null, "confidence": 0.0~1.0, "reason": "판단 근거 한 줄"}

```

## 6. 사람 검수 체크리스트

GPT-4o 결과가 나오면 민아가 아래 기준으로 검수합니다.

**우선 검수 대상 (반드시 직접 확인):**

- confidence가 0.7 미만인 건 전부
- primary_label이 0(NORMAL)인데 reason을 읽어보면 애매한 건
- secondary_label이 있는 건 (복수 패턴이면 primary가 맞는지 확인)

**검수 방법:**

- 스크린샷을 직접 열어보고, GPT가 준 reason이 화면 내용과 맞는지 확인
- 경계 사례 판단표(위 3번)와 비교해서 primary/secondary가 적절한지 확인
- 틀리면 직접 수정하고, `reviewer` 필드에 "민아" 기입

**검수 후 수정 비율 기록:**

- `검수 결과: GPT 정답률 __%, 수정 __건, 주요 오류 유형: ___`

## 8. 주의사항

**감각 조작(13번)은 보수적으로 판정합니다.** 정적 스크린샷에서는 깜빡임이나 애니메이션을 확인할 수 없으므로, 시각적으로 명백한 흔적(예: 파란색 깜빡임 영역이 캡처된 경우)이 아니면 13번으로 라벨링하지 않습니다.

**도메인에 따라 용어만 달라질 뿐 판단 기준은 동일합니다.** 금융에서 "이율·원금 손실"이면 쇼핑몰에서는 "배송비·위약금·자동갱신", 구독 서비스에서는 "무료체험 후 자동결제" 등으로 바뀔 뿐, 같은 유형이면 같은 라벨을 붙입니다.

**"이건 다크패턴인가 아닌가" 자체가 애매하면**, 일단 라벨을 붙이되 confidence를 0.5 이하로 낮게 주고, reason에 "판단 보류 - 팀 논의 필요"라고 적어둡니다. 나중에 팀 회의에서 일괄 정리합니다.
